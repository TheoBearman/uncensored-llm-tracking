# Curated search terms for the continuous tracking pipeline.
#
# This file is INTENTIONALLY DIFFERENT from safety_terms.txt — that file is the
# paper's canonical term set, kept stable for reproducibility. This file is
# the operational keyword set for ongoing weekly snapshots, optimized for
# precision (low false-positive rate) over recall.
#
# Line format: <term>[:max_results]
#   - Without a cap, the term runs uncapped (the global --max-repos still applies).
#   - Inline caps suit broad terms where the long tail is mostly noise. Example:
#       roleplay:1000
#
# Comment-out (don't delete) terms you remove, so the rationale stays visible.
